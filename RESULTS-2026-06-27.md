# VERITY verity_compare — measured results (2026-06-27)
Naive vs harness on the SAME model. Harness arm = real web search (`:online`) + 2-pass verify (cli models: gate+verify, no search). Two-axis: lift AND token cost. Small N (--fast on most), so p mostly n.s. — read the consistent DIRECTION + the coherent pattern.

## Coherent finding (confirms REQUIREMENTS.md doctrine)
**Harness lift ∝ 1 / (model strength × task easiness).** Biggest lift on weaker models + current/post-cutoff tasks; flat-or-negative on strong models + ceiling tasks (the verify pass second-guesses already-correct answers). Two-axis cost flags frontier token-blowout.

## Honest caveats
- **Frontier ceiling:** gemini/gpt-5.5 already score ~100% on these dims → no headroom → +0 lift (and huge token cost). The harness's value is on weaker models + current-info, not topping out a frontier model on easy tasks.
- **Strong-model over-verification regression:** the 2-pass verify can WORSEN a strong model on easy tasks by second-guessing a correct answer (codex −25). Fix path: skip the verify pass when the draft is high-confidence / deterministically-checkable; reserve panel+judge for genuinely high-stakes (v2.2 gate).
- **`current` dimension is the real validator:** +33.3pp proves the search-augmented harness on post-cutoff facts — a naive model literally cannot know `uv`/newest-model-IDs without search.
- cli (codex) N is tiny (4 tasks); treat the −25 as directional, not precise.
