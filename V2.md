# VERITY v2 — Harness Sovereignty Layer

> **Harness sovereignty > model quality. Build with frontier, run with open-source.**
> You own the rig; the model is a rented, swappable brain. A great harness on a local open-weights model beats a genius model used raw — and keeps running when a frontier model gets pulled mid-project.

v2 distills 2 years of a hand-built self-learning harness + the safest patterns from `safe-agentic-workflow`, `pi`, and the OpenCode / VSCodium / Venice sovereign stack into VERITY's gate engine. Everything runs **fully local, zero external keys** (Ollama), with optional private-cloud (Venice) as a tier.

## New in v2

### 🛠️ Code executor — drive OpenCode on an isolated worktree (`verity-opencode`)
VERITY now executes code through OpenCode (model-agnostic), not just talk:
```
verity-opencode plan  "<goal>"          # read-only pre-flight: plan + files-touched + RISK + falsifiable criterion
verity-opencode build "<goal>" [-m M]   # execute, JSON step events (supervisable)
verity-opencode solve "<goal>"          # plan-gate → build on an ISOLATED git worktree; HIGH-RISK auto-halts
```
Plan-agent gates risk before the build-agent touches anything; build runs in `.worktrees/` so it can't clobber main. Model swaps with `-m provider/model` or `$OPENCODE_MODEL` — e.g. `ollama/qwen2.5:3b-instruct` (local, $0) or `venice/qwen3-next-80b` (private cloud).

### 🚦 New deterministic gates (fire every session)
- **SPEC-GATE** — no real work without ≥1 *falsifiable* success criterion (AC/DoD). Ambiguous goal → label GUESS, clarify, halt. *(from safe-agentic-workflow)*
- **FRESH-CONTEXT CHECK** — significant output is verified by a NEW sub-agent in a clean adversarial context: maker ≠ checker at the *conversation* level, not just the model. The producer never grades itself. *(from the 2-yr harness)*
- **TOOL VETO** — a destructive/out-of-scope tool call returns `{block, reason}` injected back to the LLM so it *self-corrects*, instead of hard-crashing. Real human gates (password/2FA/payment/delete-prod) still defer. *(from pi `beforeToolCall`)*
- **DURABLE VERDICT** — VERIFIED/GUESS/BLOCKED + evidence persisted to an external system-of-record that survives context compaction; the never-assume gate reads it instead of re-deriving. *(from safe-agentic-workflow + pi)*

### 🧩 Reusable harness primitives
- **`commands/`** — pipeline blueprints as `.md`: phases → which agent/tool runs each → what passes between → quality threshold → terminal condition. The orchestrator *reads* the command; it doesn't re-derive the plan. Repeatable work becomes deterministic.
- **`skills/INDEX.md`** — specialist-scoped skill injection: the orchestrator reads only the index; each skill body is injected only into the sub-agent that needs it, keeping the orchestrator lean across long runs.
- **`handoff.md`** — cross-session *state* continuity (the playbook already gave discipline continuity). Next session resumes goal + phase + open failures with no memory infra.

### 🔐 Sovereign routing (R26 local-first)
`local Ollama / RotorQuant → Venice (private/uncensored, OpenAI-compatible) → OAuth-subscription shim → enterprise API (last)`. Author the harness with frontier; run it on open weights you own. Sovereign IDE: VSCodium + Continue → local/Venice (no telemetry, no cloud code-exfiltration).

## Sell points (why it matters)
- **Vendor-proof.** When a frontier model is pulled or rate-limited mid-project, the harness keeps running on local weights. The rig is the durable asset.
- **$0 marginal inference.** Runs on local Ollama out of the box; no per-token spend, no API key, no signup.
- **Private by default.** Sensitive work never leaves the machine (local) or goes only to zero-retention inference (Venice TEE).
- **Weak model, strong results.** The gates (search-before-concluding, fresh-context verify, never-quit, spec-gate, objective completion) make a 3B local model behave with frontier-grade discipline.
- **Auditable.** Every gate firing + verdict is logged to the ledger — "the harness helped" is a receipt, not a vibe.
- **Composable.** Executor + memory (mempalace, local) + semantic code search (claude-context, local) + routing (ccr) — all open-source, all keyless.
